---
title: "Theme toggle"
description: "Dark and light theme switching with localStorage persistence."
---

The portfolio supports dark and light themes. The active theme is saved to `localStorage` so the user's choice persists across browser sessions.

## How it works

A single `setTheme(mode)` function controls the theme. It adds or removes the `light-theme` class on `<body>` and updates the toggle button label.

```javascript
const themeToggle = document.getElementById('theme-toggle');
const THEME_KEY = 'portfolio-theme';

function setTheme(mode) {
  if (mode === 'light') {
    document.body.classList.add('light-theme');
    if (themeToggle) themeToggle.textContent = '🌙 Dark';
  } else {
    document.body.classList.remove('light-theme');
    if (themeToggle) themeToggle.textContent = '☀ Light';
  }
}
```

On page load, the saved theme is read from `localStorage`. If no preference has been stored, the default is `dark`.

```javascript
const savedTheme = localStorage.getItem(THEME_KEY) || 'dark';
setTheme(savedTheme);
```

The toggle button's click event reads the current state from the class list, computes the next theme, applies it, and writes the choice back to `localStorage`.

```javascript
if (themeToggle) {
  themeToggle.addEventListener('click', () => {
    const isLight = document.body.classList.contains('light-theme');
    const nextTheme = isLight ? 'dark' : 'light';
    setTheme(nextTheme);
    localStorage.setItem(THEME_KEY, nextTheme);
  });
}
```

<Note>
  The `localStorage` key is `portfolio-theme`. Once a user selects a theme, their preference is restored automatically on every subsequent visit.
</Note>

## Button labels

| Current theme | Button text |
| ------------- | ----------- |
| Dark          | `☀ Light`  |
| Light         | `🌙 Dark`   |

The label always shows the theme the button will switch **to**, not the current theme.

## CSS variables

All colors are defined as CSS custom properties on `:root`. The `body.light-theme` selector overrides each variable with lighter equivalents. No additional selectors need to change — everything adapts automatically.

```css
:root {
  --bg: #050a0e;
  --bg2: #0a1520;
  --bg3: #0d1e2e;
  --accent: #00d4ff;
  --accent2: #00ff9d;
  --text: #e8f4f8;
  --muted: #607080;
  --border: rgba(0,212,255,0.15);
  --grid-line: rgba(0,212,255,0.03);
  --scan-line: rgba(0,0,0,0.04);
}

body.light-theme {
  --bg: #f4f9ff;
  --bg2: #ffffff;
  --bg3: #edf5ff;
  --accent: #0077ff;
  --accent2: #00b894;
  --text: #0f172a;
  --muted: #4b5563;
  --border: rgba(0, 119, 255, 0.18);
  --grid-line: rgba(0, 119, 255, 0.06);
  --scan-line: rgba(0, 0, 0, 0.02);
}
```

The `body` element also carries a transition so the swap is animated rather than instant:

```css
body {
  transition: background-color 0.25s ease, color 0.25s ease;
}
```
